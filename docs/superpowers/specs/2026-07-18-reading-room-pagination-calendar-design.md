# 枕书：书架分页与批注日历设计

## 背景

GitHub Pages 版本目前一次性渲染完整书架和全部批注。当前线上数据包含 273 个书架条目和 814 条可展示批注；一次性铺开不便浏览，也会产生不必要的 DOM 节点。全部批注都有创建日期，跨越多个活跃月份，适合增加按月日历导航。

本次只增强 GitHub Pages 前端的浏览能力，保留现有“枕书”视觉样式，不修改微信读书同步接口、GitHub Actions 或 `reading-room.json` 数据格式。

## 已确认需求

- 书架采用页码分页，每页 24 本。
- 日历统计全部批注，包括划线和想法/点评。
- 点击日期后只展示当天的全部批注。
- 批注列表每页 20 条，避免初始页面一次渲染全部内容。
- 搜索继续覆盖书名、作者、分类、批注原文、想法和章节。
- 保留当前页面颜色、字体、卡片和移动端风格。

## 方案选择

采用浏览器端分页和日历筛选。完整 JSON 仍只加载一次，分页、搜索、日期分组和月份切换均在浏览器内完成。

不采用无限滚动，因为用户已选择明确页码；不把数据拆成多个分页或月份 JSON，因为当前数据量不需要额外网络分片，拆分还会增加同步脚本、缓存和发布的一致性成本。

## 页面结构

### 书架分页

书架网格保持现有四列桌面布局和两列移动布局。列表下方增加分页栏：

- 上一页按钮。
- 当前页附近的页码；页数较多时用省略号压缩中间页。
- 下一页按钮。
- “共 N 本 · 第 X/Y 页”摘要。

每页固定展示 24 个条目。首页禁用上一页，末页禁用下一页。点击页码后平滑滚动到书架标题，而不是页面最顶端。

搜索先过滤完整书架，再对结果分页。搜索词变化时书架自动回到第 1 页；清除搜索后恢复完整书架并从第 1 页显示。

### 批注日历

批注标题下、列表上方增加月历卡片：

- 顶部显示当前年月、上一月、下一月和“全部批注”。
- 主体为周一到周日的七列网格。
- 有批注的日期显示朱红色圆点和当天条数。
- 选中日期使用朱红色描边或浅色底，保持纸张质感。
- 没有批注的日期不可选或保持弱化。
- 默认月份为最新一条批注所在月份；如果没有批注，则使用当前月份。

点击有批注的日期后，批注列表只显示当天内容，标题旁显示日期和结果数。再次点击同一天，或点击“全部批注”，清除日期筛选。

日历月份切换只改变日历视图，不会自动筛掉其他月份的批注；只有明确点击某一天才启用日期筛选。这样用户可以自由查看月份，同时不会误以为内容丢失。

### 批注分页与搜索

批注列表每页 20 条，使用与书架一致的分页组件。处理顺序固定为：

1. 从全部批注中应用搜索词。
2. 如果选中了日期，再应用日期筛选。
3. 保持同步文件中的倒序时间排列。
4. 对结果分页。

搜索词或选中日期变化时，批注自动回到第 1 页。切换日历月份但未选择日期时，不重置列表。批注为空时区分“该日没有匹配内容”“没有搜索结果”和“尚未同步批注”三种提示。

## 状态与模块边界

现有 `github-pages/assets/app.js` 继续负责读取数据、绑定事件和渲染页面。新增独立的纯逻辑模块，负责：

- 计算分页总页数并截取当前页。
- 生成带省略号的页码序列。
- 按 Asia/Shanghai 时区把 Unix 时间戳转换为日期键。
- 汇总每天的批注数量。
- 生成指定月份的日历网格。

页面状态增加书架页码、批注页码、日历月份和选中日期。纯逻辑模块不访问 DOM，可直接用 Node 测试；渲染层只消费其返回结果。

## 时间口径

所有日期都按 `Asia/Shanghai` 计算，与网站现有日期展示保持一致。日期键统一使用 `YYYY-MM-DD`，月份键使用 `YYYY-MM`，避免浏览器本地时区导致批注落到前一天或后一天。

## 响应式与可访问性

- 桌面端日历与批注列表共用当前内容宽度，不增加侧栏。
- 移动端日历保持七列，隐藏非必要文字，仅保留清晰日期和数量标记。
- 页码和日期都使用原生按钮，提供可辨识的禁用、选中和键盘焦点状态。
- 日历按钮包含完整日期的无障碍标签；选中日期使用 `aria-pressed`。
- 分页切换后平滑滚动到对应列表标题，键盘焦点保留在触发分页的按钮上。

## 错误与边界处理

- 数据为空时不渲染无意义分页。
- 当前页超过过滤后的最大页数时自动收敛到最后一页。
- 缺少或无效 `createTime` 的批注不进入日历统计，但仍能在“全部批注”列表中展示。
- 日历跨年切换正常处理 12 月到次年 1 月。
- 月份没有批注时仍显示完整月历，并弱化所有日期。
- 现有 JSON 加载失败提示保持不变。

## 测试与验收

新增纯逻辑测试覆盖：

- 273 个书架条目按 24 个一页得到 12 页，末页 9 个。
- 页码序列在首页、中间页和末页正确显示省略号。
- 搜索或筛选后页码自动收敛。
- Asia/Shanghai 日期键转换正确。
- 闰年、跨年、周一开头和周日结尾月份的日历网格正确。
- 每日批注计数包含划线和想法/点评。
- 日期筛选与搜索组合后再分页。

静态页面测试确认新增日历和分页容器、脚本模块引用存在，且仍不调用 Sites 同步 API。上线前运行项目测试、lint 和生产构建。

部署后验收：

1. 书架首屏只展示 24 本，可切换到第 12 页并看到末页 9 本。
2. 搜索书名后结果和页数正确，清除搜索恢复第 1 页。
3. 批注日历默认打开最新批注月份，有内容日期显示数量。
4. 点击日期后只出现当天划线和想法；“全部批注”能恢复完整列表。
5. 桌面端和手机端分页、月历均可操作，原有视觉样式保持一致。

## 非目标

- 不新增服务端分页接口。
- 不改变 GitHub Actions 同步频率或数据内容。
- 不按天保存新的历史快照；日历仅使用现有批注的创建时间。
- 不增加编辑、删除或写回微信读书功能。
- 不重做首页、书架卡片或阅读统计区域。
