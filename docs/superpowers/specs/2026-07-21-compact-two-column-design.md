# 紧凑双栏阅读档案设计

## 目标

解决当前首页内容铺开、留白过多、日历占比过大的问题。保留现有阅读时间轴、书架分页、批注日历和每日 23:30 自动同步，不增加新的数据能力。

## 结构

- 页面主容器收窄到约 1120px，左右内边距缩小。
- 首页首屏采用主栏 + 窄侧栏：主栏展示统计和最近批注时间线；侧栏展示本月日历、当前阅读和阅读活动摘要。
- 日历改为紧凑辅助卡片：单元格、内边距和工具栏高度降低，只强调有批注日期。
- 书架与批注列表沿用现有分页与搜索能力，统一收窄卡片宽度和间距。
- 移动端退化为单列，侧栏模块按“日历 → 当前阅读 → 活动摘要”顺序堆叠。

## 视觉与交互

- 保留石纸、森林绿、琥珀色和衬线标题的视觉识别。
- 减少装饰性大标题和大面积空白，使用较小的标题、统计条和卡片间距。
- 日历仍支持月份切换、全部日期和点击日期查看批注。
- 不恢复页面手动同步入口。

## 验收

- 桌面端首屏在 1120px 左右容器内完成主要信息展示。
- 首页日历高度明显低于当前版本，并且不再与时间线争夺主视觉。
- 书架和批注页在 6 本/页、20 条/页的现有规则下保持可读。
- `npm test`、`npm run lint`、`npm run build` 通过。
