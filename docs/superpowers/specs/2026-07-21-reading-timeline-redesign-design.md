# 枕书：阅读时间轴整站重设计

## 决策

采用 C「阅读时间轴」作为整站信息架构。首页从单一书架入口改为阅读档案：先展示总览数据，再以真实批注的创建时间组成时间轴，右侧辅助展示月份日历、正在阅读和阅读统计。

## 页面结构

- 顶部导航：阅读档案、我的书架、全部批注；搜索入口保留。
- 总览首屏：藏书数、批注数、累计阅读时长，以及最近同步信息。
- 主内容：按 `createTime` 倒序排列的批注时间轴，保留书名、章节、日期、划线/想法类型和微信读书链接。
- 侧栏：批注日历、正在阅读的书、每周/月阅读统计。
- 书架视图：真实书架卡片继续每页 6 本，保留搜索和分页。
- 批注视图：保留日历按日筛选和每页 20 条，改成时间轴卡片呈现。
- 移动端：侧栏下沉到主内容后方，时间轴变为单列，导航和搜索保持可用。

## 视觉系统

- 背景使用石白纸张色，正文使用深森林绿。
- 时间轴节点、选中日期和重点数据使用琥珀橙。
- 标题使用中文衬线字体，辅助信息使用系统无衬线字体。
- 采用细边框、轻阴影和少量圆角，形成当代档案馆而非通用后台的质感。
- 所有按钮、日期和分页控制保留键盘焦点与明确的选中状态。

## 数据与行为

数据仍从 `github-pages/data/reading-room.json` 静态读取，不修改导出格式和 GitHub Actions。页面继续保留每日北京时间 23:30 定时同步；不添加页面手动同步入口。时间轴只使用已有真实 `createTime`，不虚构缺少时间依据的阅读事件。

## 验收

- 默认首页可同时看到总览、时间轴、日历/书架和统计入口。
- 点击导航可以在阅读档案、书架和批注视图之间切换。
- 日期筛选、搜索、分页和书籍链接行为不回归。
- 空数据、无日期批注和移动端布局均有合理提示。
- 运行完整测试、lint、build，并通过 GitHub Pages 发布验证。
