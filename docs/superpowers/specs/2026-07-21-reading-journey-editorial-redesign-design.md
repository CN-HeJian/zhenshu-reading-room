# 全程阅读心路：编辑部长文式前端重设计

## 目标

把现有“全程阅读心路”从多块信息卡片改成一篇可以顺着阅读的个人长文。数据、DeepSeek 分析结果、历史归档和现有站点配色保持不变，只优化信息层级、阅读节奏和响应式布局。

## 设计方向

采用“编辑部长文风（A）”：窄内容列、明显章节、连续时间轴、轻量的证据提示和折叠归档。页面依然使用现有的森林绿、琥珀色、米白纸张感和衬线标题，不新增图片依赖或图表库。

## 页面结构

1. **开篇摘要**：显示章节编号、标题、核心判断、更新时间和当前重点类别。摘要压缩为适合首屏阅读的窄幅内容，并通过一条细分隔线与正文连接。
2. **长时间轴**：四个阶段纵向排列。时间段作为左侧边注，阶段标题、正文和“基于你的批注与阅读记录”提示作为正文。每个阶段使用不同的细线/圆点节奏，但不引入视觉噪声。
3. **重点类别演化**：用水平步骤条展示“技术分析 → 趋势交易 → 价值投资 → 资产配置 → 投资心性”，下方只保留当前类别的概述和阶段性变化，避免把所有内容堆在右侧栏。
4. **三组回看内容**：反复出现的主题、发生过的转向、仍在追问，以章节小标题和两列短卡呈现；移动端自动单列。
5. **历史归档**：按日期的折叠目录置于末尾，默认只打开当前归档；每条归档展示标题、摘要和阶段缩略条，点击后再展开细节。

## 交互与可访问性

- 保留现有顶部标签导航，不引入新路由。
- 历史归档继续使用原生 `details/summary`，支持键盘操作。
- 所有分析数据仍通过现有 JSON 文件加载；缺少分析时保留清晰的空状态。
- 不依赖 JavaScript 才能理解页面结构；JS 仅负责填充数据和控制显示。

## 响应式规则

- 桌面端正文最大宽度约 1040px，减少当前页面的横向铺开感。
- 时间轴在桌面端保持左边时间边注，窄屏时切换为上方日期。
- 重点类别步骤条在窄屏时横向滚动或换行，正文内容单列。
- 归档缩略条在窄屏时两列显示，避免过高的单列列表。

## 数据与实现边界

- 不改变 `reading-journey.json`、`reading-journey-history.json` 的结构。
- 只调整 `github-pages/index.html` 的心路区域、`github-pages/assets/styles.css` 的心路样式，以及 `github-pages/assets/app.js` 的轻量渲染标记。
- 继续使用 `escapeHtml` 处理模型输出，避免把分析文本作为 HTML 执行。

## 验证标准

- 现有测试、构建和静态文件检查继续通过。
- 心路数据可用时，页面按“摘要 → 时间轴 → 类别演化 → 回看内容 → 归档”顺序阅读。
- 心路数据不可用时，空状态和历史归档仍可见。
- 桌面端不再出现大面积空白，两列内容在窄幅容器内保持紧凑；移动端无横向溢出。
