# 全程阅读心路设计

## 目标

使用 DeepSeek 对微信读书批注进行长期、宏观的阅读心路分析。输出关注阅读阶段、思想转向、反复出现的问题和最爱类别的主题迁移，不做以数字为中心的周报。

## 分析节奏

- 微信读书数据仍然每天 23:30 同步。
- DeepSeek 每周运行一次，并保留 `workflow_dispatch` 手动触发。
- 首次运行使用现有全部批注建立长期基线；后续运行使用长期记忆、历史分析摘要和新增证据增量重建全程轨迹。

## 上下文与归档

- 原始批注保留在 `reading-room.json`，不全部重复发送给模型。
- `analysis-history/reading-memory.json` 保存压缩后的阶段、转折、长期主题和未解问题。
- `analysis-history/YYYY-MM-DD.json` 保存每次完整分析。
- 网站读取 `reading-journey.json` 展示最新分析，读取 `reading-journey-history.json` 展示全部归档。
- API Key 只在 GitHub Actions Secret 中使用，不进入浏览器。

## 输出结构

- 全程判断
- 阅读阶段时间线
- 发生过的转向
- 反复出现的主题
- 最爱类别的长期变化
- 仍在追问的问题
- 证据边界说明

模型必须基于批注证据，不进行心理诊断，不在证据不足时臆测；DeepSeek 失败时保留上一版分析并继续发布网站。

## 验收

- 工作流每天同步、每周分析。
- 分析页面可以看到长期阅读轨迹和历史归档。
- 首次无分析时页面有清晰提示，不影响书架和批注页面。
- 测试、lint 和 build 全部通过。
